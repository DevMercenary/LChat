# LChat — локальный чат по локальной сети

Прямой P2P-чат между двумя компьютерами в одной локальной сети для обмена
текстом (копипаста) и файлами. Без серверов, без интернета, без аккаунтов.

- **Язык:** Rust
- **UI:** нативный (`egui`/`eframe`)
- **ОС:** Fedora Linux и Windows 11 (один и тот же код)
- **Транспорт:** прямое TCP-соединение, порт **9009**
- **Шифрование:** Noise (X25519 + ChaCha20-Poly1305 + BLAKE2s), общий пароль
- **Автопоиск:** UDP-broadcast маячок, порт **9010**
- **Предпросмотр:** картинки и текст — встроенный; видео — кадр через `ffmpeg`

## Скачать готовую сборку

Собранные бинарники (без установки Rust) — на странице
**[Releases](https://github.com/DevMercenary/LChat/releases)**. Их автоматически
собирает GitHub Actions при каждом теге версии.

- **Windows 11:** скачай `LChat-windows-x86_64.zip`, распакуй, запусти `lchat.exe`.
  При первом запуске разреши доступ к сети для **частных сетей**.
  Для значка в меню «Пуск» можно запустить `install.ps1` из архива.
- **Fedora/Linux:** скачай `LChat-linux-x86_64.tar.gz`, распакуй и запусти
  `./install.sh` (поставит в меню приложений) либо просто `./lchat`.

> Как выпустить сборку (для мейнтейнера): `git tag v0.1.0 && git push --tags` —
> Actions соберёт архивы и приложит их к релизу.

## Как это работает

Каждый экземпляр приложения одновременно:
- **слушает** порт 9009 (принимает входящие подключения);
- умеет **подключаться** к IP второго компа;
- рассылает и слушает маячки автопоиска на UDP-порту 9010.

Соединение двустороннее: подключиться должен только один из двух — дальше текст и
файлы ходят в обе стороны.

### Шифрование и пароль

Весь трафик после подключения шифруется (Noise NNpsk0). Общий **пароль**
(поле «Пароль» в окне) превращается в ключ и служит одновременно шифрованием и
защитой от чужих: если пароли на двух компах не совпадают — соединение не
устанавливается («рукопожатие не удалось»). Пароль должен быть одинаковым с обеих
сторон; пустой пароль тоже работает (но тогда подключиться сможет любой в сети —
задайте общий секрет).

### Автопоиск

Оба компьютера видят друг друга автоматически: в окне под «Мои адреса» появляется
строка «Найдены в сети:» с кнопкой на каждого соседа — нажатие сразу подключает,
IP вводить не нужно. Если автопоиск не срабатывает (Wi-Fi с изоляцией клиентов
блокирует broadcast) — используйте ручной ввод IP.

## Установка

Быстрее всего — готовыми установщиками (подробности и Windows см. в
[INSTALL.md](INSTALL.md)):

- **Fedora/Linux:** `./install.sh` — соберёт и поставит `lchat` в `~/.local/bin`
  с ярлыком в меню приложений.
- **Windows 11:** `install.ps1` (правой кнопкой → «Выполнить с помощью PowerShell»)
  — соберёт `lchat.exe`, поставит в профиль и создаст ярлык в «Пуск».

## Сборка вручную

Нужен Rust (`rustup`). В папке проекта:

```bash
cargo build --release
```

Готовый бинарник:
- Fedora: `target/release/lchat`
- Windows: `target\release\lchat.exe`

Windows-сборку можно собрать прямо на Windows (поставить rustup с rust-lang.org,
затем `cargo build --release`). Кросс-компиляция из Fedora тоже возможна
(`rustup target add x86_64-pc-windows-gnu` + `mingw`), но проще собрать на месте.

## Запуск и подключение

1. Запусти `lchat` на **обоих** компьютерах.
2. Задай одинаковый **пароль** в поле «Пароль» на обоих компах.
3. Подключись одним из способов:
   - **Автопоиск:** нажми кнопку соседа в строке «Найдены в сети:».
   - **Вручную:** посмотри IP в строке «Мои адреса» на одном компе, впиши его в
     поле «IP собеседника» на другом и нажми **Подключиться** (порт по умолчанию
     9009).
4. Готово. Текст — поле внизу, `Ctrl+Enter` или кнопка «Отправить».
   Файлы — кнопка «📎 Файл…» или просто **перетащи файлы в окно**
   (drag-&-drop): при перетаскивании окно подсветится, отпусти — уйдёт
   собеседнику.

### Предпросмотр файлов в чате

- **Картинки** (png, jpg, gif, webp, bmp…) показываются миниатюрой прямо в ленте;
  клик — открыть в системе.
- **Текстовые файлы** (txt, md, json, csv, код и т.п.) — сворачиваемый
  предпросмотр содержимого (показывается начало, до 64 КБ).
- **Видео** (mp4, mkv, webm, mov…) — кадр-миниатюра (извлекается через `ffmpeg`,
  если он установлен) и кнопка «▶ Открыть в плеере». Встроенного проигрывания
  нет — видео открывается системным плеером.

> Для миниатюр видео нужен `ffmpeg` в PATH (Fedora: `sudo dnf install ffmpeg`;
> Windows: поставить ffmpeg и добавить в PATH). Без него видео просто
> открывается кнопкой, без превью.

- Принятые файлы сохраняются в `~/LChat-received` (кнопка «🗀 Папка приёма»
  открывает её).
- У каждого текстового сообщения есть кнопка «Копировать» — удобно для копипасты.
- Вставка **текста** в поле ввода — обычный `Ctrl+V`.

### Отправка картинки/файла из буфера обмена

Скопируй картинку (например, скриншот) или файл и отправь его в чат:

- нажми кнопку **«📋 Из буфера»**, либо
- нажми **`Ctrl+V`**, когда курсор **не** в поле ввода текста (в поле `Ctrl+V`
  по-прежнему вставляет текст).

Перед отправкой появится **окно подтверждения** с предпросмотром — на случай, если
в буфере оказалось не то, что нужно. Нажми «✅ Отправить» или «Отмена».

> Поддерживается картинка из буфера (сохраняется как PNG) и **скопированные
> файлы** из проводника/файлового менеджера (можно несколько сразу).
> На Linux/Wayland для чтения списка файлов нужен `wl-clipboard`
> (`sudo dnf install wl-clipboard`), на X11 — `xclip`; картинки работают и без них.

### Автозапуск при старте системы (опционально)

Галочка **«Запускать при старте системы»** в окне добавляет/убирает LChat из
автозапуска **текущего пользователя** (права администратора не нужны):

- **Fedora/Linux:** создаётся `~/.config/autostart/lchat.desktop`.
- **Windows 11:** прописывается ключ в реестре
  `HKCU\Software\Microsoft\Windows\CurrentVersion\Run`.

Автозапуск ссылается на текущий путь к бинарнику — не перемещай файл после включения
(иначе включи галочку заново).

### История чата и настройки

Настройки (IP, пароль — по желанию) и история чата сохраняются на диск:
- Linux: `~/.config/lchat/config.json`
- Windows: `%APPDATA%\lchat\config.json`

- Галочка **«Сохранять историю чата»** — включает/выключает сохранение переписки
  (при выключении история из конфига удаляется).
- Галочка **«запоминать»** рядом с паролем — сохраняет пароль в конфиг
  (в открытом виде; по умолчанию выключено).
- Кнопка **«🧹 Очистить чат»** — стирает все сообщения из окна и из сохранённой истории.

## Файрвол (важно!)

Нужны TCP-порт 9009 (чат) и UDP-порт 9010 (автопоиск).

**Fedora (firewalld):**
```bash
sudo firewall-cmd --add-port=9009/tcp --permanent
sudo firewall-cmd --add-port=9010/udp --permanent   # для автопоиска
sudo firewall-cmd --reload
```

**Windows 11:** при первом запуске Защитник Windows спросит разрешение на доступ
к сети — нажми «Разрешить доступ» для **частных сетей**.

Оба компьютера должны быть в одной подсети (один роутер/свитч).

## Структура кода

- `src/net.rs` — сеть: слушатель, менеджер соединения, рукопожатие Noise,
  шифрованный транспорт (чанки ≤ 65535 байт), кадровый протокол
  (`[тип:1][длина:8 BE][данные]`, тип 0 = текст, 1 = файл), автопоиск,
  сохранение файлов.
- `src/main.rs` — GUI на `egui`, фоновые потоки общаются с UI через каналы.

Тесты (рукопожатие, шифрованный обмен, отказ при неверном пароле, маячок,
санитайз имён): `cargo test`.

## Возможные доработки

- Потоковая передача больших файлов (сейчас файл целиком читается в память).
- История сообщений на диск.
- mDNS как альтернатива broadcast (надёжнее в некоторых сетях).
